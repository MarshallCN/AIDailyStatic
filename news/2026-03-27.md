## 纽约公立医院不再续约 Palantir 合同
- source: The Guardian
- date: 2026-03-26
- category: 应用/产业
- url: https://www.theguardian.com/technology/2026/mar/26/new-york-hospitals-palantir-ai
- summary: 纽约市公立医院宣布 2026 年 10 月 Palantir 合同到期后不会续签，准备转用内部系统。
- detail: |
    合同主要用于帮忙追缴 Medicaid 等公共保健的差额收入，自 2023 年 11 月以来支付了将近 400 万美元，院方强调整个项目从一开始就是短期的，且在合约里写死了不会把患者数据给 ICE，Palantir 在系统里只能看到有限的账单笔记。院方还补充，合同结束后 Palantir 的应用和数据都将被移除，后续会部署完全由自己开发的营收循环平台，不再与外部公司共享资料。

    这一宣布发生在 Palantir 在英国 NHS 花 3.3 亿英镑做的数据系统正遭到审查的背景下，Medact 等健康正义团体警告该软件可能被用于扩张政府监控，英国仍有不到一半的卫健当局正式启用系统。政府方面虽在压进度，Palantir 也坚持自己不拥有客户数据，并称任何数据滥用都是违法。

## GitHub 更新 Copilot 交互数据训练策略
- source: GitHub Blog
- date: 2026-03-25
- category: 安全
- url: https://github.blog/news-insights/company-news/updates-to-github-copilot-interaction-data-usage-policy/
- summary: GitHub 让 Copilot Free/Pro/Pro+ 的交互数据在 4 月 24 日起默认用于模型训练，用户可在「隐私」设置里退订。
- detail: |
    交互数据涵盖用户接受或修改的建议、向 Copilot 发送的输入（包含代码片段）、游标周围的上下文、评论、文件名与仓库结构、功能互动、反馈评级等。GitHub 表示这些数据只在用户使用 Copilot 时才会被收集，而且企业版（Business、Enterprise）并不受此次政策影响；原本选择退订的设置依旧生效，如果未来再启用 Copilot，偏好会被保留。

    这波调整强调它并不会处理静态的私有仓库内容：「在没使用 Copilot 的情况下，你的代码依旧是安全的」。GitHub 也指出这些交互数据会和微软等集团公司共享，但不会交给第三方模型供应商，目的在于改善提议准确度、提前发现漏洞、加速多语言支持，让模型在真实工作流中表现更好。

## Anthropic CMS 误开放数千条未发布资产
- source: Fortune
- date: 2026-03-26
- category: 安全
- url: https://fortune.com/2026/03/26/anthropic-leaked-unreleased-model-exclusive-event-security-issues-cybersecurity-unsecured-data-store/
- summary: Anthropic 的内容管理系统配置失误导致近 3,000 条未发布的博客、活动和素材可以在网络上匿名访问。
- detail: |
    剑桥大学的网络安全研究员 Alexandre Pauwels 发现这些资产原本存放在 Anthropic 的 CMS 里，但默认对外可见，任何掌握请求方式的人都能列出并下载这些尚未公开的 PDF、图片和文稿。在得知该情况后，Anthropic 迅速锁定权限，直到修复后才将内容设为私有。

    公司发言人将这一事故归因于 CMS 配置的人为失误，强调这些只不过是候补的发布素材，未涉及核心基础设施、AI 系统或客户数据。事件仍提醒人们，当默认权限设为公开、且没有经过人工加固时，即便没有恶意外部攻击，也可能无意中泄露大量敏感草稿。

## Google TurboQuant 用极致压缩刷新 KV 缓存效率
- source: Google Research
- date: 2026-03-24
- category: 论文
- url: https://research.google/blog/turboquant-redefining-ai-efficiency-with-extreme-compression/
- summary: Google Research 提出 TurboQuant，将 PolarQuant 与 QJL 联手实现 3-bit KV 缓存、零精度损失并显著加速注意力计算。
- detail: |
    TurboQuant 先通过 PolarQuant 随机旋转向量，把它们映射到极坐标，以更容易预测的角度空间进行高质量压缩，再用只占 1 bit 的 Quantized Johnson-Lindenstrauss（QJL）消除残余误差，使注意力得分保持无偏。这个串联让量化常数不再暴增，并把传统方法需要的额外内存压缩掉。

    论文在 LongBench、Needle In A Haystack、ZeroSCROLLS、RULER 和 L-Eval 等长上下文基准上展示：TurboQuant 能把 KV 缓存缩小至原来的 1/6 甚至更少（3 bit 量化），而不需再训练或微调，还能在 H100 上让注意力逻辑比 32-bit 未量化版本快 8 倍，并在向量搜索 recall 指标上全面领先 PQ 与 RabbiQ，说明在降低内存成本的同时也能保持检索精度。

## Google 警告量子计算将在 2029 年破译现有加密
- source: The Guardian
- date: 2026-03-26
- category: 基础设施
- url: https://www.theguardian.com/technology/2026/mar/26/google-quantum-computers-crack-encryption-2029
- summary: Google 表示大型量子计算机可能在 2029 年前突破当前加密标准，敦促工程团队立即部署后量子密码。
- detail: |
    官方博客提到，量子计算机会在几年内成为破解主流加密的重大威胁，因此已把认证服务的威胁模型调整为优先迁移后量子密码，并呼吁同行尽快跟进，避免目前的数据在未来被「存档后再解密」的攻击回放。当今加密仍需巨量液氦和超低温稳定环境，但一旦能稳定保持数十万甚至百万个 qubit，现有机制就不堪一击。

    虽然有学者认为具备破译能力的量子机可能要到 2030 年代或 2050 年代才出现，但 Google 的警告促使政府和情报机构提前行动，英国国家网络安全中心早在 2035 年前就建议各单位做好准备。专家因此建议，将今天仍需保密的敏感文件（例如十年前的机密）用后量子方案再加一层防护，以免未来被一台量子计算机轻易恢复。

## Outworked 0.3.0 发布 Pixel Office 与多 agent 统筹框架
- source: GitHub Releases
- date: 2026-03-27
- category: 开源,生态
- url: https://github.com/outworked/outworked/releases/tag/v0.3.0
- summary: 开源项目 Outworked 0.3.0 推出 Pixel Office、内建浏览器与 Cloudflare 隧道等功能，进一步拓展 agent 生态。
- detail: |
    3 月 27 日早上 07:09 发布的 0.3.0 版本把交互式 Pixel Office 作为主界面，用户可以为每个 agent 设定角色、模型、人格与精灵图，在像素办公室里观察它们写代码、协同、发消息或执行定时任务。每个 agent 都内置 Claude Code 工具（Bash、编辑、阅读等）、完整浏览器，并可以用 tunnel_start 直接把本地服务暴露给 Cloudflare 公网。

    新版本的技能系统依旧通过 SKILL.md 声明工具、触发器和身份验证，支持 GitHub、PostgreSQL、Slack、Linear 等 MCP 服务器，也能插入权限闸门、计划任务、消息渠道与成本仪表板。该项目采用 GPL-3.0 许可，强调「本地运行、无额外 API key」，对想快速搭建可视化 agent 生态的开发者提供了可扩展的开源基础。
