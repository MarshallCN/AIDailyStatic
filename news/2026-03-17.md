## OpenAI 与 AWS 合作，将模型带入美国政府机密与非机密云环境
- source: TechCrunch
- date: 2026-03-17
- category: 应用/产业,生态,安全
- url: https://techcrunch.com/2026/03/17/openai-expands-government-footprint-with-aws-deal/
- summary: OpenAI 与 AWS 达成合作，将通过 AWS 的政府云渠道向美国政府销售可用于机密和非机密工作的 AI 产品。
- detail: |
    OpenAI 上个月刚与五角大楼达成合作，开始进入美国国防系统的机密网络环境；3 月 17 日的新动作则把这条路径从单一国防合同扩展到 AWS 的公共部门分发体系。对 OpenAI 来说，这不只是多卖一批 API，而是进入政府采购和合规部署链路。

    这意味着政府云正在成为前沿模型公司的新竞争场。模型能力之外，谁能更快嵌入既有云渠道、满足分类环境要求并建立可信交付流程，谁就更可能在公共部门拿到长期订单。后续值得关注的是，哪些机构会率先落地，以及 OpenAI 与 AWS 如何处理更高敏感级别场景下的治理边界。

## Niv-AI 走出隐身模式，瞄准 GPU 瞬时功耗与数据中心电力浪费
- source: TechCrunch
- date: 2026-03-17
- category: 应用/产业,基础设施
- url: https://techcrunch.com/2026/03/17/niv-ai-exits-stealth-to-wring-more-power-performance-out-of-gpus/
- summary: 以色列初创公司 Niv-AI 完成 1200 万美元种子轮融资，试图通过毫秒级监测与调度降低 AI 数据中心的 GPU 电力浪费。
- detail: |
    随着训练与推理集群越做越大，AI 数据中心的瓶颈正在从“有没有 GPU”转向“能不能把现有 GPU 以稳定、经济的方式跑满”。TechCrunch 援引现场发言指出，当前一些数据中心会因为电力与电网管理问题被迫把利用率压低，损失可达 30%。

    Niv-AI 的思路是先把问题量化。公司在机架级部署新型传感器，以毫秒级精度捕捉 GPU 功耗波动，并据此建立不同深度学习任务的电力画像；下一步则是做一层“电力副驾驶”，预测并协调负载，把原本被浪费的电力容量重新释放出来。公司称未来 6 到 8 个月内会在少数美国数据中心上线可运行系统。

    这类创业方向说明，AI 基础设施优化已经深入到供电与电网耦合层。对运营商和模型公司而言，新增算力不一定都来自买更多芯片，也可能来自更细粒度的能源调度与负载管理。后续要看 Niv-AI 能否把“省电”真正转化成可量化的吞吐提升与资本开支回报。

## NVIDIA 在 GTC 2026 推出本地代理栈：DGX Spark、Nemotron 3 与 NemoClaw 组合落地
- source: NVIDIA Blog
- date: 2026-03-17
- category: 基础设施,开源,安全
- url: https://blogs.nvidia.com/blog/rtx-ai-garage-gtc-2026-nemoclaw/
- summary: NVIDIA 在 GTC 2026 展示面向本地代理的新开源与安全栈，让 DGX Spark 和 RTX PC 能运行 Nemotron 3 模型与 NemoClaw。
- detail: |
    本地优先的代理系统正在变得更有吸引力，因为企业和个人都希望在隐私、成本和可控性之间找到平衡。NVIDIA 这次把“能在桌面上跑起来的代理栈”打包得更完整：硬件是 DGX Spark 与 RTX 工作站，模型是 Nemotron 3 系列，安全与运行时则由 NemoClaw 和 OpenShell 负责。

    NVIDIA 在 3 月 17 日的文章中提到，Nemotron 3 Super 是一个 1200 亿参数、120 亿激活参数的开源模型，适合复杂代理任务；同时还推出更轻量的 Nemotron 3 Nano 4B，并为 Qwen 3.5、Mistral Small 4 等开放模型提供本地优化。文章还称 Nemotron 3 Super 在面向 OpenClaw 的 PinchBench 上取得 85.6% 成绩。另一条关键信息是 NemoClaw 这套开源栈会安装 OpenShell 运行时，用于更安全地运行长期自治代理。

    这组发布表明，代理基础设施的竞争正在从“云上最强模型”延伸到“本地可运行、成本可控且有安全边界的全栈方案”。如果 DGX Spark 和 RTX 生态能把这些能力稳定下放到开发者桌面，本地代理开发门槛会继续下降。后续值得关注的是，社区对 NemoClaw 的采用速度，以及这些本地方案在真实企业工作流中的稳定性。

- date: 2026-03-17

- summary: Tools for Humanity 发布 AgentKit 测试版，希望用 World ID 与 x402 协议证明 AI 购物代理背后有唯一真实用户授权。
- detail: |
    “代理购物”正在从演示走向实际部署，但一旦 AI 代理可以代替用户在网站上浏览、下单和支付，平台马上会遇到新的欺诈、刷单和垃圾流量问题。World 背后的 Tools for Humanity 想把自己定位成这一波代理商业化的信任层。

    TechCrunch 报道称，AgentKit 测试版允许网站通过 World ID 来验证某个购物代理是否代表一位真实且唯一的人行动。它把 World 的身份体系接入到 Coinbase 与 Cloudflare 推动的 x402 协议，让网站在接受代理发起的交易时，能够判断背后是否存在经过验证的人类授权。当前这项能力面向开发者测试，且用户需要先完成基于 Orb 的 World ID 验证。

    如果代理电商继续升温，身份验证、支付协议与网站侧风控会成为关键基础设施，而不是边缘配件。World 这次的机会在于卡住“真人授权”这一节点，但挑战同样明显：生物特征验证的接受度、开发者接入成本以及平台是否愿意把信任层外包给单一供应商，都还需要观察。

## RelayCaching 论文提出复用解码期 KV Cache，加速多代理 LLM 协作
- source: arXiv
- date: 2026-03-17
- category: 论文,基础设施
- url: https://arxiv.org/abs/2603.13289
- summary: 出现在 3 月 17 日 arXiv 新列表的 RelayCaching 论文提出在多代理协作中复用解码期 KV Cache，可将首 token 延迟最高降低 4.7 倍。
- detail: |
    多代理 LLM 系统常常会在多个代理之间重复传递相似上下文，导致 prefill 阶段一遍遍重算，既浪费显存，也拖慢响应速度。RelayCaching 关注的正是这个瓶颈：不是继续优化单模型推理，而是降低代理协作过程中的重复计算。

    论文提出直接复用前一个代理在解码阶段产生的 KV Cache，并只对少量受前缀影响的位置做选择性重算。作者报告称，在数学推理、通识问答和代码生成等协作任务上，这种方法能实现超过 80% 的 KV Cache 复用率，并把首 token 延迟最多压低到基线的约四分之一，同时保持较小的精度损失。

    这类工作对代理框架和推理平台都很有价值，因为行业正在从“一个大模型单打独斗”转向“多个角色分工协作”。如果 RelayCaching 这类方法能稳定落地，未来多代理系统的成本结构和交互延迟都有机会明显改善。

## ILION 论文尝试为代理执行前置加“确定性安全闸门”
- source: arXiv
- date: 2026-03-17
- category: 论文,安全,基础设施
- url: https://arxiv.org/abs/2603.13247
- summary: 出现在 3 月 17 日 arXiv 新列表的 ILION 论文提出一种面向代理动作执行前的训练无关安全闸门，用于在操作发生前做 ALLOW 或 BLOCK 判定。
- detail: |
    当 AI 代理开始操作文件系统、调用 API、修改数据库甚至发起交易时，传统文本审核模型已经不够用了，因为问题不再只是“这段话是否有害”，而是“这个动作是否越权”。ILION 试图把安全控制前移到执行之前，把动作授权本身变成一个独立系统层问题。

    论文提出一个不依赖额外训练数据和外部 API 的五级级联架构，在代理真正执行动作前给出 ALLOW 或 BLOCK 的确定性判断。作者在自建的 ILION-Bench v2 上报告了 0.8515 的 F1、91.0% 的精确率和 143 微秒平均延迟，并与 Lakera Guard、OpenAI Moderation API、Llama Guard 3 做了对比。不过这套结果目前主要来自作者自建基准，仍需要更广泛复现。

    它的意义在于把“代理安全”从内容过滤推进到执行治理。如果未来代理系统普遍需要通过类似的前置安全闸门，企业部署自主系统时的架构重点也会随之变化。后续观察点是基准是否开放、业界是否能复现这些结果，以及这类确定性机制能否覆盖更复杂的真实工作流。
