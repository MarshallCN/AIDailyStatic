## Anthropic 限制 Claude Code 与第三方工具整合
- source: Tech in Asia
- date: 2026-04-05
- category: 应用/产业
- url: https://www.techinasia.com/news/anthropic-to-restrict-claude-code-use-with-third-party-tools
- summary: Anthropic 表示 Claude Code 订阅原本没为第三方整合设计，从 4 月起收紧这类用法。
- detail: |
    Tech in Asia 报道 Boris Cherny 在 X 上说明，Claude Code 目前的订阅层级没有针对外部 harness 产生的使用模式设计，因此要限制那些在第三方平台自动触发的大量调用，让流量回归到原本的服务期望之内。

    此前 TechCrunch 也披露 Anthropic 已要求使用 OpenClaw 等第三方工具的订阅用户另行付费，意味着这些整合必须在官方产品与外部生态之间重新划拨预算，而不能继续凭单一订阅混用。

    这波收紧促使开发者重新评估跨平台架构——要么直连 Claude Code，避免额外成本，要么转向自托管或替代模型，才能掌控长期的可预测性。

## 古吉拉特邦高等法院为司法系统划定 AI 红线
- source: Tech in Asia
- date: 2026-04-04
- category: 安全
- url: https://www.techinasia.com/news/indias-gujarat-high-court-restricts-ai-in-judicial-decisions
- summary: 古吉拉特邦高院允许 AI 做辅助但严禁用于正式裁决，所有输出必须人工复核。
- detail: |
    Tech in Asia 报道该法院在新准则中限定 AI 只能参与行政、法律检索和 IT 自动化等非裁判性任务，虽可加速准备材料，却不能代替人类判官的推理。

    重要的是，任何 AI 生成的草稿都必须交由人类官员复核、核对出处与逻辑，确保没有未经核实的信息直接进入判决书，从而保留最终责任在人工之手。

    此举表明印度司法系统更倾向守住安全边界：让 AI 承担重复性工作、提升效率，但在关键裁决环节坚持人为控制，避免因模型偏差或不可解释性引发信任危机。

## Google Research 发布 TimesFM 时间序列基础模型
- source: GitHub Trending
- date: 2026-04-05
- category: 论文
- url: https://github.com/google-research/timesfm
- summary: Google Research 推出面向时间序列预测的 TimesFM，借助预训练结构提升多行业趋势精度。
- detail: |
    TimesFM（Time Series Foundation Model）是 Google Research 将基础模型理念首次拓展到时间序列领域的成果，训练目标覆盖金融、能源、制造等行业常见的时序信号，借助大规模预训练获取跨域的模式理解力。

    项目随同代码与 checkpoint 一起在 GitHub 开源，配套的说明文档强调可与 TensorFlow 推理流水线对接，并提供不同长度窗口的示例，让团队不需从零构建即可迅速试验。

    初始版本意在降低时间序列任务的入门门槛：以预训练建模取代繁重特征工程，使研究人员把精力放在因果解释和场景适配，而非每天调参。

## LLM Wiki：让 LLM 自我维护的知识底座
- source: Hacker News
- date: 2026-04-04
- category: 生态
- url: https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f
- summary: LLM Wiki 让代理在 Markdown 维基中持续编写更新，形成持续演化的知识资产。
- detail: |
    与传统每次调用都重新检索的 RAG 模式不同，LLM Wiki 由代理承担持久知识库的维护工作：每当新资料出现，它会更新实体页面、生成摘要并标出矛盾点，让信息在维基内累积。

    人类仍负责挑选信源与校验关键事实，但 LLM 处理结构化写作、内部链路与冲突提醒，使知识演进不再是一次性任务，而是长期的、可追踪的协作项目。

    这种模式让生态团队能把时间花在判断优先级上，而非重复抓取，促使代理与人类共同守护一套不断增长的“知识底座”。
