## OpenAI 收购科技脱口秀 TBPN，把媒体分发纳入自身体系
- source: TechCrunch
- date: 2026-04-02
- category: 应用/产业,生态
- url: https://techcrunch.com/2026/04/02/openai-acquires-tbpn-the-buzzy-founder-led-business-talk-show/
- summary: OpenAI 收购硅谷科技脱口秀 TBPN，但保留其独立品牌与日常节目运作，把“媒体分发”正式纳入自身传播体系。
- detail: |
    TechCrunch 报道称，这是 OpenAI 首次收购媒体公司。TBPN 全名为 Technology Business Programming Network，由前创业者 John Coogan 和 Jordi Hays 主持，长期在 YouTube 和 X 上做三小时直播，覆盖科技、商业、AI 与国防议题。

    按照报道，TBPN 会继续保留自身品牌，同时向 OpenAI 首席政治与传播负责人 Chris Lehane 汇报。节目的吸引力不只在流量，还在它已经成为硅谷创始人、投资人和大厂高管之间一种“业内人对业内人”的表达场。

    这笔交易释放出的信号很直接：头部 AI 公司开始把传播渠道本身当成战略资产。未来围绕 AI 的竞争，不只是模型、算力和分发接口，也包括谁能更主动地塑造行业叙事。

## ElevenLabs 推出 ElevenMusic，正式下场做 AI 音乐应用
- source: TechCrunch
- date: 2026-04-02
- category: 应用/产业
- url: https://techcrunch.com/2026/04/02/elevenlabs-releases-a-new-ai-powered-music-generation-app/
- summary: ElevenLabs 发布 iOS 应用 ElevenMusic，支持用提示词生成、混音和发现 AI 音乐，免费层每天可生成 7 首歌。
- detail: |
    ElevenLabs 过去的品牌重心一直是语音模型，但这次上线 ElevenMusic，意味着它开始更明确地往“面向消费者的完整创作应用”走。用户可以直接通过自然语言提示词生成歌曲，也能对别人创作的歌曲做 remix。

    产品设计上，ElevenMusic 兼顾了创作和消费两端。免费版每天可生成 7 首歌，可调节时长、歌词和风格；Pro 订阅则提供每月 500 首曲目配额、更大存储空间以及更多风格能力。它同时还内置直播电台、情绪歌单和类似流媒体平台的浏览方式。

    这说明 ElevenLabs 不打算只做底层语音供应商，而是想提前占住“AI 音频消费入口”。对创意 AI 赛道来说，下一阶段的竞争将越来越像“模型 + 工具 + 分发”的组合战。

## Microsoft 一次性补齐转写、语音与图像三类基础模型
- source: TechCrunch
- date: 2026-04-02
- category: 应用/产业,基础设施
- url: https://techcrunch.com/2026/04/02/microsoft-takes-on-ai-rivals-with-three-new-foundational-models/
- summary: Microsoft AI 一口气发布 MAI-Transcribe-1、MAI-Voice-1 和 MAI-Image-2，继续补齐自家的多模态基础模型栈。
- detail: |
    这三款模型分别瞄准语音转写、音频生成和图像生成。按照微软对外说法，MAI-Transcribe-1 覆盖 25 种语言，速度约为 Azure Fast 方案的 2.5 倍；MAI-Voice-1 可以在 1 秒内生成 60 秒音频；MAI-Image-2 则从 MAI Playground 进一步走向更正式的对外分发。

    更关键的是，这批模型来自 Mustafa Suleyman 领导的 Microsoft AI / MAI Superintelligence 团队。也就是说，微软虽然仍与 OpenAI 深度绑定，但正在持续搭自己的多模态底座，避免在关键产品层完全依赖外部能力。

    对微软来说，这不只是“再发几个模型”，而是在 Foundry、Playground 和自家产品体系里铺一条可独立演进的模型管线。对外部开发者而言，这会让微软在价格、集成和部署层面拥有更大谈判空间。

## Google Vids 让用户直接用提示词指挥头像“说话和行动”
- source: Google Workspace Updates
- date: 2026-04-02
- category: 应用/产业,生态
- url: https://workspaceupdates.googleblog.com/2026/04/direct-avatar-to-speak-and-act-anywhere-with-a-consistent-face-and-voice.html
- summary: Google Vids 新增可定制头像与动作提示，让用户通过文字指令控制同一张脸和声音在不同场景里说话、走动和使用物体。
- detail: |
    这项更新把 Google Vids 里的 AI 视频能力往前推进了一大步。用户现在不仅能选择固定 speaker，还能上传产品图、办公室图或额外参考图，让头像在不同场景里保持一致的脸和声音。

    更重要的是，Vids 不再只是“选模板、换文案”。Google 明确写到，用户可以直接输入 prompt，让头像走动、说话、操作物体，甚至围绕某个商品或办公室场景完成演示。这让它更接近一个轻量级、面向商务场景的可控视频生成工具。

    这类产品化路径会显著降低视频制作门槛，尤其适合培训、内部公告和销售演示。对办公软件生态而言，AI 角色视频正在从“新奇功能”变成文档、表格和幻灯片之外的新一层默认输出格式。

## Gemini API 新增 Flex 与 Priority 两档推理层
- source: Google AI Blog
- date: 2026-04-02
- category: 基础设施,观察
- url: https://blog.google/innovation-and-ai/technology/developers-tools/introducing-flex-and-priority-inference/
- summary: Google 为 Gemini API 增加 Flex 和 Priority 两档推理层，试图让开发者在成本、延迟与稳定性之间做更细粒度的取舍。
- detail: |
    Google 在官方博客里把这次更新定义得很直接：为 Gemini API 增加两种新的 inference tier，用来平衡 cost、latency 和 reliability。也就是说，模型 API 的产品设计已经不再只是“给你一个 endpoint”，而是开始像云服务一样细分不同服务等级。

    从开发者视角看，这种分层很有现实意义。某些批处理、异步分析或价格敏感的任务可以接受更大的延迟波动，另一些线上交互和关键生产流则需要更稳定的优先保障。Google 现在是在把这类现实差异显式产品化。

    这条更新很值得当作趋势看待，因为 API 竞争正在从“谁的模型更强”延伸到“谁能把算力调度和成本曲线包装得更适合生产环境”。参考来源：Google AI Blog。

## NVIDIA 把 Gemma 4 推向 RTX PC、DGX Spark 与边缘设备
- source: NVIDIA Blog
- date: 2026-04-02
- category: 基础设施,开源
- url: https://blogs.nvidia.com/blog/rtx-ai-garage-open-models-google-gemma-4/
- summary: NVIDIA 宣布把 Gemma 4 优化到 RTX PC、DGX Spark 和 Jetson Orin Nano 等平台，推动更强的本地 agentic AI 部署。
- detail: |
    NVIDIA 把这次动作定义为“从 RTX 到 Spark”的一次完整铺开。公司表示，Google 与 NVIDIA 已协同把 Gemma 4 针对自家 GPU 做了更深的优化，让这些开放模型可以跨数据中心、工作站、个人 AI 超算和边缘模块部署。

    按官方介绍，Gemma 4 家族覆盖 E2B、E4B、26B 和 31B 等多个变体，重点强调 reasoning、coding、原生 function calling 以及视觉、视频、音频等多模态能力。它所面向的场景，不再只是“本地跑一个聊天模型”，而是更偏向能接本地上下文和工具链的 agentic workflow。

    这件事的重要性在于，开放模型与本地 GPU 的结合正在从极客玩法走向标准部署选项。只要端侧与边缘设备能吃下更强的多模态模型，很多原本默认上云的工作流都会重新评估部署边界。

## H Company 发布 Holo3，继续把“computer use”代理推向企业工作流
- source: Hugging Face
- date: 2026-04-02
- category: 论文,开源
- url: https://huggingface.co/blog/Hcompany/holo3
- summary: H Company 在 Hugging Face 发布 Holo3，声称以 10B active / 122B total 参数在 OSWorld-Verified 上做到 78.85%，主打企业桌面“computer use”代理。
- detail: |
    Holo3 这次强调的不是单点 demo，而是一套面向企业环境的训练方法。团队称其通过“agentic learning flywheel”在合成企业环境里反复学习执行流程，从而把模型从通用界面操作推进到更贴近真实业务系统的任务集合。

    官方还给出了一套更贴近企业落地的验证方式，包括自动生成的 Synthetic Environment Factory，以及包含 486 个多步任务的 H Corporate Benchmarks，覆盖电商、业务软件、协作和多应用混合场景。与此同时，Holo3-35B-A3B 权重也以 Apache 2 许可证在 Hugging Face 开放。

    这条路线值得关注，是因为它说明开放“computer use”代理正在从浏览器演示转向更复杂的企业软件环境。如果这类模型继续提升，未来自动化不只是写代码或回邮件，而是直接跨系统完成多步业务操作。
