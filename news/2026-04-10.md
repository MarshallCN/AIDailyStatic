## Claude Managed Agents 把 agent 运行时进一步平台化
- source: Anthropic Docs
- date: 2026-04-10
- category: 基础设施,生态
- url: https://platform.claude.com/docs/en/managed-agents/overview
- summary: Anthropic 在 4 月上旬开放 Claude Managed Agents Beta，把 agent harness、环境、会话和事件流做成托管式基础设施。
- detail: |
    从文档可见，Claude Managed Agents 不只是一个模型 API，而是一整套运行时：开发者可以预配置环境、权限、工具与会话，让 Claude 在托管容器内长时间异步执行任务。

    这意味着 agent 竞争正在从“模型能不能做”进一步转向“谁来提供生产级执行基础设施”。当 sandboxes、状态管理、权限边界和长时运行都被平台接管后，企业接入门槛会明显降低，但对平台的依赖也会更深。

## 观察：agent 时代的控制权开始从“模型能力”转向“运行边界”
- source: Anthropic Engineering
- date: 2026-04-10
- category: 观察
- url: https://www.anthropic.com/engineering/managed-agents
- summary: Anthropic 在工程文章中提出“把大脑和手分开”，反映出 agent 竞争正从模型本身转向 harness、权限与运行环境的控制权。
- detail: |
    文章强调，随着模型能力上升，旧的 harness 假设会迅速过时，因此需要把运行时抽象做得更稳定、更可托管，让平台负责工具执行、上下文管理与长时任务支撑。

    结合 OpenClaw 的接入波动和 OpenAI 面临的现实诉讼，可以看到一个共同趋势：真正决定 agent 能否大规模落地的，已经不只是“会不会做事”，而是“谁定义它能做什么、在哪做、出事时谁负责”。

    参考来源：Anthropic Engineering《Scaling Managed Agents: Decoupling the brain from the hands》、TechCrunch《Anthropic temporarily banned OpenClaw’s creator from accessing Claude》。
