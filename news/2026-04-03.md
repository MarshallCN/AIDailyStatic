## Sony Interactive Entertainment 收购 Cinemersive Labs，加码视觉计算
- source: Sony Interactive Entertainment
- date: 2026-04-02
- category: 应用/产业
- url: https://sonyinteractive.com/en/news/blog/sony-interactive-entertainment-acquires-cinemersive-labs/
- summary: SIE 收购英国的 Cinemersive Labs，把它并入 Visual Computing Group，以增强机器学习驱动的游戏画面能力。
- detail: |
    Sony Interactive Entertainment 表示，PlayStation 旗下的视觉计算探索正在推进，他们已与英国机器学习与计算机视觉创业公司 Cinemersive Labs 达成收购协议，目的是推进更沉浸、超越现实的画面体验。

    Cinemersive Labs 自 2022 年成立，由拥有计算机视觉与机器学习专项能力的团队组成，完成交割后将融入 SIE 的 Visual Computing Group（VCG），并参与渲染、画面增强、游戏视觉 fidelity 等项目的长期研发。

    SIE 期待借助 Cinemersive 的专业能力，不断提升 PlayStation 上的视觉表现和玩家体验，把《OC》内部力量重新整合，为下一阶段的算力+AI 美术打下更高台阶。

## Google Meet 登上 CarPlay，大幅提升出行中会议接入
- source: Google Workspace Updates

- detail: |

## OpenAI 细说 Model Spec 背后的理念
- source: OpenAI
- date: 2026-04-02
- category: 论文
- url: https://openai.com/index/our-approach-to-the-model-spec/
- summary: OpenAI 解读 Model Spec 的结构与价值，把模型行为的预期从内部原则转化为对外可读的治理框架。
- detail: |
    Model Spec 诞生于 OpenAI 对公平、安全与普惠的信念，它不是声称模型已经完美，而是用一套可公开检验、可持续演进的行为准则为未来训练、评估与算法治理定方向。

    文章把 Model Spec 看成与 Preparedness Framework、AI Resilience 等计划并行的三叉治理：前者应对能力风险，后者提高社会承受力，而 Model Spec 则聚焦日常指令执行、冲突调解与用户自由之间的平衡。

    这份规范展开了由三大目标（赋能开发者/用户、防止严重伤害、维护运营许可）构成的序言，再衔接更细化的行为指导；不同的行为类型需要不同的写法，所以里头同时包含宏观意图、具体决策路径以及随实地部署不断修改的条款。

## OpenAI 推出 Safety Bug Bounty，补足滥用/安全边界
- source: OpenAI
- date: 2026-04-02
- category: 安全
- url: https://openai.com/index/safety-bug-bounty/
- summary: OpenAI 在 Bugcrowd 上推出安全漏洞计划，专注发现 AI 滥用与安全风险而非传统的技术漏洞。
- detail: |
    这项公测的 Safety Bug Bounty 与已有的 Security Bug Bounty 并行，重点在于可复现实验性的 agentic 滥用（如 prompt 注入、自动 Agent 替代投稿、脱敏信息泄露），以及可能造成物理或社会危害的行为。

    计划罗列了几类风险：入侵 agent 之後窃取资料、模型透露专有逻辑、操纵账号与平台信号等；排名、权限绕过等问题仍建议报送给 Security Bug Bounty，但整套机制已同步交由 Safety 与 Security 团队共同分类。

    文章指出，若研究者能交付可操作的危害原型与修复思路也可能获赏，鼓励安全、伦理黑客与研究社群持续在 Bugcrowd 上提交案例，强化 AI 生态的防护网。

## Google 推出 Gemma 4 开放模型家族
- source: Google AI Research
- date: 2026-04-02
- category: 开源
- url: https://blog.google/innovation-and-ai/technology/developers-tools/gemma-4/
- summary: Google 发布 Gemma 4 家族（E2B/E4B/26B MoE/31B Dense），以 Apache 2.0 授权公开最强开源模型。
- detail: |
    Gemma 4 以“更高的 intelligence-per-parameter”作为卖点，大型 MoE 与 Dense 模型在 Arena.ai 排名分别达到第 3 与第 6，声称比已有开源模型少 20 倍参数就能抵近竞品表现。

    家族涵盖面向边缘（Effective 2B/4B）以及高性能服务器（26B MoE/31B Dense）的版本，前者强调离线、多模态、低延迟与与 Android/Jetson 的整合，后者则支持 80GB H100 原始 bfloat16、量化版本能跑在消费者 GPU 上。

    Google 强调 Gemma 4 全部以 Apache 2.0 授权发布，并在训练、部署与安全机制上与专有模型一样审核，期望开发者能自行微调、在本地部署，并借此推动 Gemmaverse 社区继续迭代。

## 观察：公开治理与开放模型正同步展开
- source: OpenAI & Google
- date: 2026-04-02
- category: 观察
- url: https://openai.com/index/our-approach-to-the-model-spec/
- summary: Openness 与治理并驾齐驱，Model Spec 与 Gemma 4 同日公布说明顶尖实验室正在把行为公开化与能力开放化视为同一张路线图的一部分。
- detail: |
    Model Spec 把预期行为、权衡声明与演进流程写成可供公众参阅的大纲，不再把安全或公平当作内部机密，而是与外部开发者、监管者共享。

    与此呼应，Gemma 4 把高性能模型与 Apache 2.0 许可证结合，既保证了开发者可以在边缘部署，又让更多人参与调优；这种让“如何用”与“如何按照期望表现”都透明化的组合，正在重新定义可信 AI 的底层路径。
