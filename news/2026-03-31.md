## Ruka-v2补足手腕与指间自由改善操控
- source: arXiv
- date: 2026-03-27
- category: 论文
- url: https://arxiv.org/abs/2603.26660v1
- summary: Ruka-v2 在原有肌腱驱动 humanoid 手基础上加入双自由度手腕与指间外展，实现更贴近人类的操控扩展，并开源所有设计文件。
- detail: |
    Ruka-v2 在 Ruka 的 11 个自由度基础上新增一个独立的 2DOF 手腕以及指间外展，使手能在受限空间完成弧形抓取、物体旋转与细线书写等动作，所有零件与控制软件都以 GitHub 链接公开。

    论文指出 Ruka-v2 在远程遥控任务中平均完成时间下降 51.3%、成功率提升 21.2%，并在 13 个复杂任务中完成双臂与单臂操作测试，还展示了 3 个自主策略学习任务的收敛与鲁棒性。

    团队为机器人学习者准备了 3D 打印与组装文档、控制固件与示范视频（https://ruka-hand-v2.github.io/），鼓励社区在抓取、多任务与仿人学习方面展开新实验。

## LiteLLM供应链遭TeamPCP攻击
- source: DreamFactory Blog
- date: 2026-03-24
- category: 安全
- url: https://blog.dreamfactory.com/the-litellm-supply-chain-attack-a-complete-technical-breakdown-of-what-happened-who-is-affected-and-what-comes-next
- summary: TeamPCP 利用 Trivy 与 Checkmarx 的被攻陷凭据，在 3 月 24 日向 PyPI 发布包含窃取密钥的 LiteLLM 1.82.7/1.82.8，影响量级接近 97M 月下载。
- detail: |
    攻击者先后在 3 月 19 日侵入 Trivy、21 日侵入 Checkmarx CI/CD，随后用窃到的令牌推送 LiteLLM 1.82.7 与 1.82.8 到 PyPI，版本各自通过 import hook 与 .pth 机制在解释器启动时执行恶意代码，致使所有 pip install litellm 的环境均可被感染。

    恶意载荷通过 litellm_init.pth 与 proxy_server.py，遍历 SSH、AWS、GCP、Azure、Kubernetes、Docker、Git、Shell 历史、.env、OpenAI/Anthropic API 密钥与云元数据等多个敏感目录与变量，再把压缩后的 tpcp.tar.gz 经 AES+RSA 加密后上传到 models.litellm.cloud，并尝试在检测到 K8s 凭证时横向扩散与植入持久化后门。

    追踪表明 LiteLLM 依赖的 600 多个开源项目与大量 CI/CD 管线可能在无版本锁定的情况下自动拉取恶意版本，建议立刻查找 litellm_init.pth、清理 pip缓存、排查 models.litellm.cloud 的访问日志并限制与 LiteLLM 相关的凭据，以弥合供应链链条的信任缺口。

## Mistral Small 4统一推理、编程与多模态能力
- source: Mistral AI
- date: 2026-03-16
- category: 开源
- url: https://mistral.ai/news/mistral-small-4
- summary: Mistral Small 4 以 Apache 2.0 开源、119B 参数的 MoE 架构，把 Magistral、Pixtral 与 Devstral 的能力合并，提供可动态调节 reasoning_effort 的多模态推理与编码能力。
- detail: |
    Small 4 引入 128 个专家、每个 token 激活 4 个专家的 Mixture-of-Experts 架构，支持 256k 上下文、原生图像与文本输入，并允许用户用 reasoning_effort 参数在低延迟模式与深度推理之间切换，实现一个模型覆盖常规对话、推理作文、编程代理等场景。

    Mistral 披露 Small 4 在延迟优化部署中完成请求时间降低 40%、吞吐量提升 3 倍；在 AA LCR、LiveCodeBench 等基准中与 GPT-OSS 120B 性能持平甚至略优，同时输出长度显著更短，降低推理成本与延迟。

    该模型在 NVIDIA HGX H100/H200、DGX B200 等硬件上推荐部署，并已与 vLLM、llama.cpp、SGLang、Transformers 等项目集成，Mistral 还以 Nemotron Coalition 成员身份推动开源 Frontier 模型的优化落地。
